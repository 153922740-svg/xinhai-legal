## 1. 协作原则

### 1.1 核心原则

```
1. 任务透明 —— 所有人知道谁在做什么
2. 响应及时 —— 任务交接不超过 30 分钟
3. 信息完整 —— 传递任务时上下文齐全
4. 质量兜底 —— 上一环节对下一环节负责
5. 问题升级 —— 30 分钟解决不了立即升级
```

### 1.2 强制规则

- ❌ 禁止直接修改他人代码
- ❌ 禁止不经过审查直接合并
- ❌ 禁止不测试就交付
- ❌ 禁止不带上下文就移交任务

## 5. 协作效率度量

### 5.1 关键指标（KPI）

| 指标 | 当前值 | 目标值 | 测量方法 |
|------|--------|--------|---------|
| 任务交接响应时间 | 45 分钟 | ≤30 分钟 | 从发出到确认 |
| 任务完成周期 | 2 天 | ≤1.5 天 | 从分配到交付 |
| 任务交接次数/任务 | 2 次 | ≤1.5 次 | 按任务跟踪 |
| 协作问题升级率 | 15% | ≤10% | 升级数/总任务 |
| 跨 Agent 满意度 | 70% | ≥85% | 月度调研 |

### 5.2 协作效率评分公式

```
协作效率分 = 
    (30 - avg_response_min) / 30 × 30    // 响应速度 30%
    + (1.5 / cycle_days) × 30            // 完成周期 30%
    + (80 / satisfaction) × 20            // 满意度 20%
    + (100 - escalate_rate) / 100 × 20    // 问题升级率 20%
```

## 6. 问题升级流程

### 6.1 升级路径

```
层级 1: Agent 之间自行协调（30 分钟内）
    ↓ 无法解决
层级 2: COO 介入协调（1 小时内）
    ↓ 无法解决
层级 3: 总裁决策（2 小时内）
```

### 6.2 升级触发条件

- 同一问题反复 3 次以上
- 预估完成时间超过截止时间
- 涉及跨团队重大决策
- 发现安全隐患或数据风险

## 7. 协作奖惩机制

### 7.1 加分项

| 行为 | 加分 |
|------|------|

| 提前完成任务 | +3 分/次 |
| 协作满意度评分优秀 | +2 分/次 |

| 行为 | 扣分 |
|------|------|
| 任务交接不带上下文 | -5 分/次 |
| 超过响应时限 | -3 分/次 |
| 未经确认擅自跳过协作环节 | -10 分/次 |
| 同一问题反复 3 次 | -10 分/次 |
