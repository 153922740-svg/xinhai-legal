# 心海法律 AI - 接口评审制度

**版本**: V1.0  
**生效日期**: 2026-05-18  
**负责人**: COO  
**执行范围**: 所有新增/修改的 API 接口

---

## 一、评审目的

1. **确保接口路径统一** - 所有小程序调用接口使用 `/api/v1/*` 前缀
2. **保证接口质量** - 参数、响应、错误码符合规范
3. **避免路径冲突** - 防止出现接口路径不一致问题
4. **提升协作效率** - 前后端对接顺畅，减少沟通成本

## 四、评审内容

### 4.1 接口路径评审

**检查清单**:
- [ ] 小程序调用路径是否为 `/api/v1/模块名/功能`
- [ ] 后端 Phase 路由是否有 v1 兼容层
- [ ] 路径命名是否清晰、无歧义
- [ ] 是否与现有接口冲突
- [ ] Nginx 配置是否需要同步更新

**示例**:
```markdown
✅ 正确：/api/v1/auth/send_sms
❌ 错误：/api/v4/sms/send（小程序无法调用）
```

---

### 4.2 接口参数评审

**检查清单**:
- [ ] 请求方法是否正确（GET/POST/PUT/DELETE）
- [ ] 必填参数是否明确标注
- [ ] 参数类型是否定义（string/number/boolean/object）
- [ ] 参数长度/范围限制是否合理
- [ ] 是否有默认值
- [ ] 敏感参数是否加密

**示例**:
```json
{
  "phone": {
    "type": "string",
    "required": true,
    "pattern": "^1[3-9]\\d{9}$",
    "description": "手机号"
  },
  "code": {
    "type": "string",
    "required": true,
    "length": 6,
    "description": "验证码"
  }
}
```

---

### 4.3 接口响应评审

**检查清单**:
- [ ] 响应格式是否统一（code/data/message）
- [ ] 成功响应是否包含必要数据
- [ ] 错误响应是否有明确错误码
- [ ] 错误信息是否友好
- [ ] 是否包含分页信息（列表接口）

**标准格式**:
```json
{
  "code": 200,
  "data": {...},
  "message": "success"
}
```

---

### 4.4 错误码评审

**检查清单**:
- [ ] 是否使用标准错误码
- [ ] 错误码是否有明确含义
- [ ] 错误信息是否对用户友好
- [ ] 是否记录详细日志（内部）

**标准错误码**:
| 错误码 | HTTP 状态码 | 说明 |
|--------|-----------|------|
| success | 200 | 成功 |
| auth_required | 401 | 需要登录 |
| auth_expired | 401 | Token 过期 |
| permission_denied | 403 | 权限不足 |
| invalid_params | 400 | 参数错误 |
| not_found | 404 | 资源不存在 |
| server_error | 500 | 服务器错误 |

---

### 4.5 安全性评审

**检查清单**:
- [ ] 是否需要认证
- [ ] 是否需要权限控制
- [ ] 是否有 SQL 注入风险
- [ ] 是否有 XSS 风险
- [ ] 是否有频率限制
- [ ] 敏感数据是否加密

---

### 4.6 文档评审

**检查清单**:
- [ ] 接口文档是否更新
- [ ] 是否包含请求示例
- [ ] 是否包含响应示例
- [ ] 是否包含错误码说明
- [ ] 是否标注小程序调用路径
- [ ] 是否标注后端实际路径

---

## 五、评审模板

### 5.1 接口设计文档模板

```markdown

- **请求方法**: GET/POST/PUT/DELETE

- **负责人**: {开发者}
- **提交日期**: 2026-XX-XX

## 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| param1 | string | 是 | 参数说明 |

## 响应格式
```json
{
  "code": 200,
  "data": {},
  "message": "success"
}
```

## 错误码
| 错误码 | 说明 |
|--------|------|
| success | 成功 |
| invalid_params | 参数错误 |

## 后端路由
- **v1 路径**: `/api/v1/模块名/功能`
- **实际路径**: `/api/vX/...`
- **处理函数**: `function_name()`

## 测试用例
- [ ] 正常场景测试
- [ ] 异常场景测试
- [ ] 边界条件测试

## 评审记录
- [ ] COO 初审：{日期} {意见}

- [ ] 前端评审：{日期} {意见}
- [ ] 测试评审：{日期} {意见}

```

## 七、违规处理

### 7.1 违规场景

- ❌ 未经评审直接开发
- ❌ 评审未通过擅自上线
- ❌ 接口路径不符合规范
- ❌ 接口文档未同步更新

### 7.2 处理措施

| 违规程度 | 处理措施 |
|----------|----------|
| 轻微 | 口头警告 + 立即整改 |
| 一般 | 书面警告 + 重新评审 |
| 严重 | 通报批评 + 绩效扣分 |
