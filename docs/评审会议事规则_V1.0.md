# 心海法律 AI - 评审会议事规则

**版本**: V1.0  
**生效日期**: 2026-05-18  
**主持人**: COO  
**适用范围**: 所有接口评审会、代码评审会、设计评审会

---

## 一、评审会类型

| 类型 | 频率 | 参与人 | 时长 | 内容 |
|------|------|--------|------|------|
| **开发前评审** | 按需 | COO+铸基 + 匠心 + 铁壁 | 30-60 分钟 | 接口设计文档评审 |
| **开发后验收** | 按需 | COO+铸基 + 匠心 + 铁壁 | 30-60 分钟 | 现场演示 + 实际测试 |

| **周评审会** | 每周五 15:00 | 全体 | 60 分钟 | 周工作总结、下周计划 |

## 二、评审会组织

### 2.1 会前准备

**开发者（提交人）**:
- [ ] 提前 24 小时提交评审材料（接口设计文档/代码/测试报告）
- [ ] 准备演示环境（本地服务启动）
- [ ] 准备测试用例和数据
- [ ] 预约评审会时间

**主持人（COO）**:
- [ ] 确认评审材料完整
- [ ] 通知参会人员
- [ ] 预定会议室/线上会议
- [ ] 准备评审记录表

**评审人**:
- [ ] 提前阅读评审材料
- [ ] 准备评审意见
- [ ] 准时参会

---

### 2.2 会议流程

**开发前评审流程**（30-60 分钟）:

```
1. 开发者介绍（5 分钟）
   - 接口功能说明
   - 技术实现方案
   - 风险评估

2. 评审人提问（10-20 分钟）
   - 铸基：技术架构、路由设计
   - 匠心：前端调用、参数格式
   - 铁壁：测试用例、验收标准

3. 评审讨论（10-20 分钟）
   - 评审人发表意见
   - 开发者回应
   - 形成评审结论

4. 评审结论（5 分钟）
   - 通过
   - 有条件通过（需修改：______）
   - 不通过（重新设计）

5. 签字确认
   - 评审人签字
   - 开发者签字
   - 主持人签字
```

**开发后验收流程**（30-60 分钟）:

```
1. 开发者演示（10 分钟）
   - 现场运行接口
   - 展示测试结果
   - 演示功能流程

2. 评审人实测（10-20 分钟）
   - 评审人亲自测试
   - 验证功能是否正常
   - 检查性能是否达标

3. 评审讨论（10-20 分钟）
   - 评审人发表意见
   - 开发者回应
   - 形成验收结论

4. 验收结论（5 分钟）
   - 通过（可以合并）
   - 有条件通过（需修复：______）
   - 不通过（重新开发）

5. 签字确认
   - 评审人签字
   - 开发者签字
   - 主持人签字
```

---

### 2.3 评审结论

**通过**:
- 所有评审项符合要求
- 测试全部通过
- 可以直接合并/上线

**有条件通过**:
- 存在小问题，不影响整体功能
- 需在规定时间内修复（24/48/72 小时）
- 修复后由 COO 验证即可，无需再次评审

**不通过**:
- 存在严重问题
- 需要重新设计/开发
- 重新提交评审

## 三、评审争议解决

### 3.1 争议类型

| 争议类型 | 解决方式 | 决策人 |
|----------|----------|--------|
| 技术分歧 | 技术评审提出方案，集体讨论 | 铸基 |
| 设计分歧 | 前端评审提出方案，集体讨论 | 匠心 |
| 质量分歧 | 测试评审提出标准，集体讨论 | 铁壁 |
| 进度分歧 | 评估风险，调整计划 | COO |
| 重大分歧 | 上报总裁决策 | 总裁 |

### 3.2 决策流程

```
评审人提出分歧 → 集体讨论（10 分钟） → 无法达成一致 → 对应负责人决策 → 仍有异议 → COO 决策 → 重大分歧 → 总裁决策
```

## 五、评审纪律

### 5.1 参会要求

- **准时**: 提前 5 分钟到场
- **准备**: 提前阅读评审材料
- **专注**: 会议期间不看手机
- **建设性**: 提出问题和解决方案

### 5.2 缺席处理

- **请假**: 提前向 COO 请假
- **委托**: 委托他人代为评审
- **补审**: 缺席评审人需单独补审

### 5.3 违规处理

| 违规行为 | 处理措施 |
|----------|----------|
| 无故缺席 | 口头警告 |
| 未准备评审意见 | 口头警告 |
| 评审不认真导致问题 | 书面警告 |
| 故意刁难 | 通报批评 |

### 6.1 时间控制

- **开发前评审**: 30-60 分钟

- **每日站会**: 15 分钟

- 提前提交评审材料（至少 24 小时）

- 会议聚焦核心问题

- 主持人严格控制时间
