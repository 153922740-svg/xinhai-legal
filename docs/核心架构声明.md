## 🔧 开发规范

### 必须遵循的 Hermes 规范

1. **使用 delegate_task 分配任务**
   - 不能单打独斗，必须通过子 Agent 执行
   - 最多同时 3 个子 Agent 并行

2. **context 必须完整**
   - 传递给子 Agent 的信息必须包含：
     - 服务器路径
     - API 地址
     - 具体要求

3. **技能使用**
   - 复杂任务前必须加载相关技能
   - 技能路径：`/home/admin/.hermes/skills/`

4. **记忆系统**
   - 用户偏好、环境事实、工具 quirks 必须保存到 memory
   - 任务进度、会话结果不保存到 memory

5. **Cronjob 使用**
   - 周期性任务使用 cronjob 调度
   - 任务必须自包含，无当前会话上下文
