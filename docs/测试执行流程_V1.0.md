## 二、测试执行流程

### 2.1 L1 单元测试（开发者执行）

**执行时机**: 开发完成后，提交评审前

**执行流程**:
```
1. 开发者编写单元测试
2. 运行单元测试
3. 修复失败用例
4. 测试通过率 100%
5. 提交测试报告
```

**测试内容**:
- 函数逻辑测试
- 边界条件测试
- 错误处理测试

**输出**: `tests/unit_test_XXX.py` + 测试报告

---

### 2.2 L2 集成测试（测试官执行）

**执行时机**: 模块开发完成，L1 测试通过后

**执行流程**:
```
1. 测试官编写集成测试用例
2. 部署测试环境
3. 执行集成测试
4. 记录测试结果
5. 提交测试报告
```

**测试内容**:
- 模块间接口测试
- 数据库操作测试
- API 接口测试

**输出**: `tests/integration_test_XXX.py` + 测试报告

---

### 2.3 L3 E2E 测试（测试官执行）

**执行时机**: 部署前必须执行

**执行流程**:
```
1. 测试官准备测试环境
2. 执行 E2E 测试用例（4 大核心流程）
3. 记录测试结果
4. 生成测试报告
5. COO 验收
```

**必测流程**:
- 登录流程（验证码发送、登录、Token 验证）
- 会员购买流程（套餐列表、订单创建、支付）
- 文书生成流程（模板列表、文书生成、下载）
- AI 对话流程（发送消息、会话列表、历史查询）

**输出**: `tests/e2e_test.py` + E2E 测试报告

---

### 2.4 L4 性能测试（测试官执行）

**执行时机**: 重大版本上线前

**执行流程**:
```
1. 准备性能测试环境
2. 执行压力测试
3. 执行并发测试
4. 记录性能数据
5. 生成性能测试报告
```

**测试内容**:
- 核心接口响应时间（<2s）
- 并发用户支持数
- 系统稳定性（72 小时）

**输出**: 性能测试报告

## 八、测试纪律

### 8.1 测试要求

- **真实性**: 测试必须真实执行，不得造假
- **完整性**: 测试用例必须覆盖完整
- **及时性**: 测试必须按时完成
- **准确性**: 测试结果必须准确记录

### 8.2 违规处理

| 违规行为 | 处理措施 |
|----------|----------|
| 测试造假 | 通报批评 + 绩效扣分 |
| 测试不完整 | 口头警告 + 重新测试 |
| 测试超时 | 口头警告 |
| Bug 漏报 | 书面警告 |
