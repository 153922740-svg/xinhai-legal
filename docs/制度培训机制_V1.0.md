# 心海法律 AI - 制度培训机制

**版本**: V1.0  
**生效日期**: 2026-05-18  
**负责人**: COO  
**适用范围**: 所有新成员、在职成员

---

## 一、培训目标

- **理解制度**: 理解所有管理制度的内容和要求
- **掌握流程**: 掌握开发流程和工具使用
- **通过考核**: 通过制度考核才能上岗
- **持续学习**: 定期复习，持续改进

---

## 二、培训对象

| 对象 | 培训类型 | 培训时间 | 考核要求 |
|------|----------|----------|----------|
| **新成员** | 入职培训 | 入职 1 周内 | 必须通过 |
| **在职成员** | 定期培训 | 每月 1 次 | 必须参加 |
| **转岗成员** | 转岗培训 | 转岗前 | 必须通过 |

---

## 三、培训内容

### 3.1 新成员入职培训

**第一天**:
- 公司介绍
- 团队介绍
- 工作环境配置
- 制度文档发放

**第二天**:
- 开发管理制度学习
- 接口管理制度学习
- 测试管理制度学习

**第三天**:
- Git 工作流学习
- 文档管理规范学习
- 工具使用培训

**第四天**:
- 实操练习
- 问题答疑
- 模拟评审

**第五天**:
- 制度考核
- 考核通过上岗
- 考核不通过补考

### 3.2 在职成员定期培训

**每月培训内容**:
- 制度更新内容
- 典型案例分享
- 经验教训总结
- 问题答疑

**培训形式**:
- 线上会议
- 文档学习
- 案例讨论
- 实操演练

### 3.3 转岗培训

**培训内容**:
- 新岗位制度
- 新岗位流程
- 新岗位工具

**考核要求**:
- 通过新岗位考核
- 完成实操练习

## 五、培训方式

### 5.1 线上培训

**工具**: 线上会议系统

**流程**:
```
培训通知 → 参加培训 → 学习材料 → 问题答疑 → 考核
```

### 5.2 线下培训

**地点**: 会议室

**流程**:
```
培训通知 → 现场培训 → 实操练习 → 问题答疑 → 考核
```

### 5.3 自学

**材料**: 制度文档、培训 PPT、实操手册

**要求**:
- 制定学习计划
- 完成学习内容
- 记录学习问题
- 参加培训答疑

## 六、考核机制

### 6.1 考核类型

| 考核类型 | 考核方式 | 合格标准 | 补考机会 |
|----------|----------|----------|----------|
| 入职考核 | 笔试 + 实操 | ≥80 分 | 2 次 |
| 定期考核 | 笔试 | ≥60 分 | 1 次 |
| 转岗考核 | 笔试 + 实操 | ≥80 分 | 2 次 |

### 6.2 考核内容

**笔试内容**:
- 制度知识（40%）
- 流程理解（30%）
- 案例分析（30%）

**实操内容**:
- 环境配置
- 代码提交
- 评审会参与
- 测试执行

### 6.3 考核题库

**制度知识题**:
1. 开发管理制度有几个阶段评审？
2. 接口设计文档包含哪些内容？
3. 36 项自检清单包括哪些方面？

**流程理解题**:
1. 描述完整的开发流程
2. 描述评审会流程
3. 描述部署流程

**案例分析题**:
1. 发现 Bug 后如何处理？
2. 评审不通过怎么办？
3. 部署失败如何回滚？

## 八、培训纪律

### 8.1 培训要求

- **准时**: 提前 5 分钟到场
- **专注**: 培训期间不看手机
- **积极**: 积极参与讨论
- **认真**: 认真学习，做好笔记

### 8.2 考勤管理

- **请假**: 提前向 COO 请假
- **迟到**: 迟到超过 10 分钟记为缺勤
- **缺勤**: 缺勤需补培训

### 8.3 违规处理

| 违规行为 | 处理措施 |
|----------|----------|
| 无故缺席培训 | 口头警告 + 补培训 |
| 考核不通过 | 补考 + 口头警告 |
| 补考不通过 | 书面警告 + 重新培训 |
| 培训作弊 | 通报批评 |

### 9.3 持续改进

- 根据评估结果改进培训内容
