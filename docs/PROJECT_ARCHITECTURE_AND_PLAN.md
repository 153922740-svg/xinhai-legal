## 四、开发流程规范

### 4.1 标准开发流程

```
[需求提出] → [PRD 评审] → [技术方案] → [设计评审]
     ↓
[开发环境编码] → [单元测试] → [代码审查]
     ↓
[测试环境部署] → [集成测试] → [测试报告]
     ↓
[预发布验证] → [上线审批] → [生产部署]
     ↓
[线上监控] → [问题追踪] → [版本发布]
```

### 4.2 Git 工作流

```
main        - 生产分支（受保护）
  ↑
develop     - 开发分支
  ↑
feature/*   - 功能分支
  ↑
hotfix/*    - 紧急修复
```

### 4.3 文档要求

每次开发必须更新：
- [ ] PRD 文档
- [ ] API 文档
- [ ] 测试用例
- [ ] 变更记录
