**后端路由**: `/api/v4/sms/send`  
**状态**: ✅ 已完成

**请求**:
```json
{
  "phone": "13800138000"
}
```

**响应**:

## 📝 七、接口开发规范

### 7.1 新增接口流程

1. 在对应 Phase 文件中实现业务逻辑
2. 添加 v1 兼容路由（Blueprint）
3. 在 app.py 中注册 Blueprint
4. 更新本文档
5. 测试验证

### 7.2 接口命名规范

- 小程序统一使用 `/api/v1/模块名/功能` 格式
- 后端 Phase 使用 `/api/vX/...` 格式（X=2,3,4,6,7,9...）
- v1 兼容路由直接调用原 Phase 处理函数

### 7.3 错误码规范

| 错误码 | HTTP 状态码 | 说明 |
|--------|-----------|------|
| success | 200 | 成功 |
| auth_required | 401 | 需要登录 |
| auth_expired | 401 | Token 过期 |
| permission_denied | 403 | 权限不足 |
| invalid_params | 400 | 参数错误 |
| not_found | 404 | 资源不存在 |
| server_error | 500 | 服务器错误 |
