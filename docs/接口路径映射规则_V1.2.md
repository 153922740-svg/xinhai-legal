## 🔧 添加 v1 兼容路由的步骤

### 步骤 1: 在 Phase 文件中添加 v1 Blueprint

```python
# 在 phaseX_xxx_api.py 文件末尾添加

# ============== v1 兼容路由（小程序专用） ==============
phaseX_v1_bp = Blueprint('phaseX_v1', __name__, url_prefix='/api/v1/模块名')


@phaseX_v1_bp.route('/接口路径', methods=['POST'])
def v1_xxx_function():
    """兼容小程序调用：功能描述"""
    return 原函数名()
```

### 步骤 2: 在 app.py 中注册 v1 Blueprint

```python
# 在 app.py 中找到对应 Phase 的注册位置
try:
    from phaseX_xxx_api import phaseX_bp, phaseX_v1_bp
    app.register_blueprint(phaseX_bp)
    app.register_blueprint(phaseX_v1_bp)
    print("✅ Phase X API 已注册 (含 v1 兼容路由)")
except Exception as e:
    print(f"⚠️ Phase X API 未加载：{e}")
```

### 步骤 3: 重启后端服务

```bash
# 终止旧进程
ps aux | grep "app.py" | grep python | grep -v grep | awk '{print $2}' | xargs kill -9

# 启动新进程
cd /home/admin/xinhai_legal_api
/home/admin/xinhai_legal_api/venv/bin/python app.py
```

### 步骤 4: 测试验证

```bash

curl -s -X POST "http://localhost:8081/api/v1/模块名/接口" \
  -H "Content-Type: application/json" \

```

## ⚠️ 注意事项

1. **所有新增接口必须提供 v1 兼容路由**
2. **v1 路由函数直接调用原 Phase 处理函数，不重复实现逻辑**
3. **接口文档必须同时标注小程序调用路径和后端实际路径**
4. **修改 Phase 处理函数时，需同步测试 v1 兼容路由**
5. **禁止直接修改 Phase Blueprint 前缀，必须通过 v1 兼容层**
