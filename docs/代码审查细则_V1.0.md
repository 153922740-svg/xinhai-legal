## 四、审查意见

### 4.1 意见分类

| 分类 | 说明 | 处理要求 |
|------|------|----------|
| **必须修改** | 严重问题，影响功能或安全 | 必须修复才能合并 |
| **建议修改** | 改进建议，不影响功能 | 建议修复，可选 |
| **疑问** | 需要 clarification | 需要回复说明 |
| **认可** | 代码写得好 | 点赞鼓励 |

### 4.2 意见格式

```markdown
## 审查意见

### 必须修改
1. **文件**: xxx.py:10
   **问题**: SQL 注入风险
   **建议**: 使用参数化查询
   ```python
   # ❌ 错误
   sql = f"SELECT * FROM users WHERE id = {user_id}"
   
   # ✅ 正确
   sql = "SELECT * FROM users WHERE id = ?"
   cursor.execute(sql, (user_id,))
   ```

### 建议修改
1. **文件**: xxx.py:20
   **问题**: 函数过长
   **建议**: 拆分为多个小函数

### 疑问
1. **文件**: xxx.py:30
   **问题**: 为什么使用这个算法？

### 认可
1. **文件**: xxx.py:40
   **认可**: 错误处理写得很好！
```

### 4.3 审查结论

**通过**:
- 所有"必须修改"已修复
- 可以合并

**有条件通过**:
- 存在"建议修改"
- 可以合并，但需要后续修复

**不通过**:
- 存在严重问题
- 需要重新开发

## 五、审查工具

### 5.1 自动化工具

- **flake8**: 代码规范检查
- **black**: 代码格式化
- **mypy**: 类型检查
- **bandit**: 安全扫描
- **pytest**: 测试执行
- **coverage**: 测试覆盖率

### 5.2 CI 检查

**GitHub Actions 配置**:
```yaml
name: Code Review CI

on: [pull_request]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Python
        uses: actions/setup-python@v2
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run flake8
        run: flake8 .
      - name: Run mypy
        run: mypy .
      - name: Run bandit
        run: bandit -r .
      - name: Run pytest
        run: pytest tests/ -v
      - name: Coverage
        run: coverage run -m pytest && coverage report
```

## 六、审查纪律

### 6.1 审查人要求

- **公正性**: 公正客观，对事不对人
- **专业性**: 专业审查，提出建设性意见
- **及时性**: 在规定时限内完成审查
- **细致性**: 仔细审查，不遗漏问题

### 6.2 提交人要求

- **配合性**: 积极配合审查，及时修复
- **谦虚性**: 虚心接受意见，不抵触
- **及时性**: 及时修复问题，不拖延
- **完整性**: 修复完整，不遗漏

### 6.3 违规处理

| 违规行为 | 处理措施 |
|----------|----------|
| 审查不认真 | 口头警告 + 重新审查 |
| 审查超时 | 口头警告 |
| 故意刁难 | 书面警告 |
| 修复不认真 | 口头警告 + 重新修复 |
| 态度恶劣 | 通报批评 |
