## 🎯 核心流程

### 开发流程（两阶段评审）

```
填写接口设计文档 → 开发前评审通过 → 开发实现 → 36 项自检 → 开发后验收通过 → 代码合并
```

### 评审流程

```
开发前评审（文档评审） → 开发后验收（现场演示 + 实际测试）
```

### 测试流程

```
L1 单元测试（开发者） → L2 集成测试（测试官） → L3 E2E 测试（测试官） → L4 性能测试（测试官）
```

### Bug 处理流程

```
发现 Bug → 记录 Bug → 分配负责人 → 修复 Bug → 验证修复 → 关闭 Bug
```

### 部署流程

```
开发环境 → 测试环境 → 预发布环境 → 生产环境
```

## 🔴 红线（违反=严重失误）

1. **未经评审先开发** - 书面警告 + 重新评审
2. **未经测试先上线** - 通报批评 + 绩效扣分
3. **擅自修改生产** - 严重警告 + 权限回收
4. **文档不同步** - 口头警告 + 立即整改
5. **自检清单未填写** - 口头警告 + 立即补填
6. **现场演示测试失败** - 重新开发 + 重新评审
7. **Bug 隐瞒不报** - 通报批评 + 绩效扣分
8. **培训考核不通过** - 补考 + 口头警告
